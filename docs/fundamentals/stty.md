# Terminal Modes & stty

<p class="page-tagline">Raw mode, canonical mode, and the kernel TTY discipline</p>

*This page is under construction.*

The `stty` command controls how the kernel's TTY line discipline processes input and output between your shell and terminal. Raw mode gives applications direct access to every keypress; canonical (cooked) mode provides line editing and signal handling.

## Key Concepts

- **Raw mode**: Every keypress sent immediately to the application. No line editing, no echo, no signal characters. What TUI apps use.
- **Canonical mode** (cooked): Input buffered until Enter. Line editing (backspace, Ctrl+U). The shell's default mode.
- **Echo**: Whether typed characters are displayed. Disabled in raw mode; enabled in canonical.
- **Signal characters**: Ctrl+C (SIGINT), Ctrl+Z (SIGTSTP), Ctrl+\ (SIGQUIT). Generated by the kernel, not the terminal.

## Common Commands

```bash
stty raw      # Switch to raw mode
stty -raw     # Back to canonical
stty -echo    # Disable echo
stty sane     # Reset to sensible defaults (recovery trick)
stty -a       # Show all current settings
```
